# Local HTML CMS

一个轻量级的本地 HTML 页面管理工具。它可以扫描 `content/` 目录下的 `.html` / `.htm` 文件，在浏览器里预览、编辑源码，也可以直接在预览区域里修改页面文字并保存回原文件。

项目适合用来管理一批静态 HTML 页面，例如活动页、落地页、简单官网页面或本地交付稿。

## 功能特性

- 自动扫描 `content/` 目录中的 HTML 文件
- 支持文件搜索、预览和源码编辑
- 支持在预览 iframe 中直接编辑页面文字
- 保存前自动检测文件是否被外部修改，避免误覆盖
- 每次保存前自动把旧版本备份到 `backups/`
- 无第三方依赖，只需要 Node.js 即可运行
- 端口被占用时会自动尝试后续端口

## 目录结构

```text
html-cms/
├─ app/
│  ├─ index.html      # 前端页面
│  ├─ main.js         # 前端交互逻辑
│  └─ styles.css      # 页面样式
├─ content/           # 放置要编辑的 .html / .htm 文件
├─ backups/           # 保存时自动生成的历史备份
├─ package.json
├─ server.js          # 本地 HTTP 服务和文件 API
└─ README.md
```

> `content/` 和 `backups/` 是运行时目录。空目录上传到 GitHub 时默认不会被保留，首次运行服务会自动创建它们。

## 环境要求

- Node.js 18 或更高版本
- 现代浏览器，例如 Chrome、Edge、Firefox

本项目没有 npm 依赖包，因此通常不需要执行 `npm install`。

## 本地运行

```bash
npm run dev
```

启动后终端会输出访问地址，默认是：

```text
http://localhost:3000
```

如果 `3000` 端口已被占用，服务会自动尝试 `3001`、`3002` 等后续端口，并在终端显示实际地址。

## 使用方法

1. 把需要管理的 `.html` 或 `.htm` 文件放入 `content/` 目录。
2. 运行 `npm run dev` 启动本地服务。
3. 在浏览器打开终端输出的地址。
4. 在左侧文件列表中选择 HTML 文件。
5. 使用右侧预览、源码编辑或“预览编辑”功能修改内容。
6. 点击“保存”，系统会先备份旧版本，再写入新内容。

## 文件与备份说明

服务只允许读取和编辑 `content/` 目录内的 `.html` / `.htm` 文件，不能通过接口修改项目其他文件。

每次保存时，旧文件会被写入 `backups/` 目录，文件名包含原路径和时间戳，例如：

```text
page.html.20260620-184500.html
folder__page.html.20260620-184500.html
```

如果保存时发现文件在磁盘上已经被其他程序修改，页面会提示是否强制覆盖。

## 常用命令

```bash
# 启动本地编辑器
npm run dev
```

## 注意事项

- 这是本地编辑工具，不建议直接暴露到公网。
- 预览区使用 iframe 加载 HTML，页面自身脚本可能会运行。
- “预览编辑”主要适合修改页面文字，不适合复杂结构调整；复杂修改建议切换到源码编辑。
- 当前服务绑定到 `0.0.0.0`，同一局域网内的设备可能可以访问，请在可信网络中使用。
